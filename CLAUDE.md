# 2freedom — A股ETF中短期交易策略

## 项目概述

基于技术指标的A股ETF量化扫描与回测系统。通过akshare接口获取全市场ETF行情，计算技术指标并打分，生成买卖信号，同时提供回测引擎验证策略有效性。

## 运行方式

```bash
pip install -r requirements.txt
python main.py
```

输出：控制台打印扫描报告 + 回测绩效，日志写入 `etf_scan.log`。

## 文件结构

```
config.py          全局配置（评分权重、止损止盈、API参数等）
indicators.py      技术指标计算（MA/MACD/RSI/布林带/ATR）
strategy.py        策略核心（加权评分、市场环境判断、信号生成）
backtest.py        回测引擎（日线模拟、绩效指标计算）
main.py            入口（并发数据获取、扫描、输出）
etf_buy_signal.py  原始版本（保留参考）
requirements.txt   依赖：akshare, pandas, numpy
```

## 策略逻辑

### 数据源
- **akshare** 接口：`fund_etf_spot_em` 获取全市场ETF实时行情，`fund_etf_hist_em` 获取单只ETF日线历史
- 筛选条件：剔除货币/债券/商品/跨境ETF，市值>2亿，日成交额>5000万，取流动性前100只
- 历史数据动态拉取最近120个自然日

### 技术指标
| 指标 | 用途 |
|------|------|
| MA5/10/20/60 | 趋势方向、均线排列、金叉判断 |
| MACD (DIF/DEA/柱状图) | 中短期动量确认 |
| RSI14 (Wilder's EMA) | 超买超卖判断 |
| 布林带 (中轨/上轨/下轨) | 波动率与支撑位 |
| ATR14 | 波动率衡量，用于动态止损止盈计算 |
| 成交量MA5 | 量能确认 |

### 评分体系（总分100）
| 信号 | 权重 | 条件 |
|------|------|------|
| MACD金叉 | 20 | DIF上穿DEA |
| 站上20日线 | 15 | 收盘价 > MA20 |
| 均线多头排列 | 15 | MA5 > MA10 > MA20 |
| MA金叉 | 15 | MA5上穿MA10 |
| RSI健康 | 15 | RSI在30~65之间 |
| 放量上涨 | 10 | 收盘上涨且成交量 > 5日均量×1.2 |
| 布林带支撑 | 10 | 价格在中轨之上、上轨之下 |

### 信号阈值
- **强烈买入**: 评分 >= 75（熊市提高至85）
- **买入观察**: 评分 >= 60（震荡市提高至65，熊市提高至75）
- **卖出**: 评分 <= 40

### 市场环境过滤
- 基于上证指数（sh000001）的MA20/MA60位置和20日动量综合评分
- 牛市（>=0.7）/ 震荡（0.3~0.7）/ 熊市（<=0.3）
- 熊市自动提高买入门槛，减少假突破风险

## 回测引擎

### 退出机制（基于ATR动态计算，非固定比例）
每只标的根据自身波动率（ATR）计算专属止损止盈价位，高波动ETF止损更宽，低波动ETF止损更紧。

1. **止盈** 买入价 + 3×ATR 平仓（盈亏比 > 1.5）
2. **止损** 买入价 - 2×ATR 平仓（约2天正常波动空间）
3. **移动止损** 盈利超过1×ATR后启动，从最高价回撤1.5×ATR平仓
4. **信号卖出** 评分降至40以下平仓

扫描输出示例：
```
  510050 | 50ETF | 评分: 85 | 价格: 2.535 | 强烈买入
    止损: 2.415 (-4.7%)  止盈: 2.715 (+7.1%)  移动止损启动: 2.555 (+0.8%)
```

### 仓位管理
- 最大同时持仓 5 只
- 单只仓位不超过总资金 20%
- 整手买入（100股为单位）

### 绩效指标
- 总收益率、年化收益率、最大回撤、夏普比率（无风险利率3%）
- 胜率、总交易次数、平均持仓天数
- 完整交易记录（含退出原因）

## 配置调整

所有参数集中在 `config.py`，可直接修改：
- 评分权重 `SCORE_WEIGHTS` — 调整各信号贡献
- 止损止盈倍数 `STOP_LOSS_ATR_MULT` / `TAKE_PROFIT_ATR_MULT` — 风险偏好（ATR倍数）
- 移动止损倍数 `TRAILING_STOP_ATR_MULT` / `TRAILING_ACTIVATE_ATR_MULT`
- 信号阈值 `BUY_THRESHOLD` / `STRONG_BUY_THRESHOLD` — 选股严格度
- ETF池大小 `ETF_POOL_SIZE` — 扫描范围

## 依赖

- Python >= 3.10（使用 `X | None` 类型语法）
- akshare（A股数据接口）
- pandas / numpy（数据处理）

## 注意事项

- akshare API依赖网络，频率过高可能被限流，并发数控制在10
- `stock_zh_index_daily` 列名可能因akshare版本不同而变化（close/open/high/low）
- 回测基于日线收盘价，不考虑盘中滑点和手续费
- 策略仅供研究参考，不构成投资建议
